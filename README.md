## 系統概覽

DesignSync 是一套專為包裝設計流程設計的線上校稿平台，解決設計師與客戶之間稿件往返混亂的問題。

**核心功能：**
- 多版本圖面管理（PDF / JPG / PNG 上傳）
- 三階段 SOP 規範清單（強制過關機制）
- 圖釘（Pin）與框選（Box）標註留言
- 版本 Diff 差異比對
- 測量工具（距離、角度）
- EmailJS 通知系統
- Firebase 雲端儲存（跨裝置同步）

## 角色與權限

| 角色 | 說明 | 主要權限 |
|------|------|---------|
| **設計師** | 上傳設計稿、管理 SOP 清單、推進流程 | 上傳檔案、打 SOP checklist、送簽推進 Stage、留言 |
| **客戶** | 校審設計稿、標註意見 | 框選/圖釘留言、核准 Stage 2 推進、結案 |
| **觀察者（老闆）** | 唯讀瀏覽 | 查看稿件與留言、刪除整個專案 |

> **Email 白名單**：各角色登入須使用授權的 Email，才能進入系統。

## 登入方式

1. 開啟系統後進入登入頁
2. 輸入**顯示名稱**（選填）與**聯絡 Email**（必填）
3. 點選對應角色卡片（設計師 / 客戶 / 觀察者）
4. 系統驗證 Email 白名單後進入儀表板

## 專案儀表板

登入後進入**專案總覽頁**，顯示所有進行中的專案。

### 功能
- **新增專案**（設計師限定）：點右上角「新增專案」，輸入名稱即可建立
- **清理空間**（設計師）：清除該專案所有檔案的舊版本，僅保留最後一版
- **刪除專案**（觀察者）：徹底刪除整個專案（不可逆）
- **結案**（客戶）：將專案標記為「已結案」並鎖定

### 專案卡資訊
- 階段進度（Stage 1 / 2 / 3）
- 目前版本號
- 狀態標籤（校稿中、等待設計師、等待客戶審核等）
- 執行天數

### 上傳新檔案
在專案內點擊「＋ 新增圖面或PDF」上傳新設計檔案，支援 PDF（自動轉圖預覽）、JPG、PNG。

## 校稿工作區

點擊檔案卡片進入工作區，分為三個區域：

```
┌─────────────────────────────────────────────────────┐
│  頂部工具列（版本切換、工具按鈕、動作按鈕）               │
├──────────────────────────────┬──────────────────────┤
│                              │  右側面板             │
│     設計圖預覽區域             │  ┌ 討論與留言 Tab    │
│  （可標註圖釘/框選、測量）      │  └ SOP 規範 Tab     │
│                              │                      │
└──────────────────────────────┴──────────────────────┘
```

### 頂部工具列
| 按鈕 | 說明 |
|------|------|
| ← 返回 | 回到專案儀表板 |
| 版本清單 | 點擊版本號切換查看各版本稿件 |
| 測量工具 | 開啟/關閉距離測量模式 |
| 更新設計版 | 上傳新版本（設計師） |
| 分享連結 | 複製連結給客戶或老闆 |
| 送出審核通知 / 完成校稿 | 依角色與 Stage 顯示不同動作按鈕 |

## SOP 工作流程（3 階段）

每個設計檔案都要走完三個 Stage 才能結案：

```
Stage 1（設計師）          Stage 2（客戶）           Stage 3（設計師）
確認規格 & 預檢    ──→    校對 & 審核準備   ──→    最終核准 & 物流
打完 Checklist           打完 Checklist            結案製作
```

### Stage 1：確認規格、檔案預檢與版本控制
- **操作角色**：設計師
- 設計師上傳稿件後，打勾所有 Stage 1 規範清單
- 完成後點「**2. 無意見（送簽給客戶, 進入 Stage 2）**」
- 若有討論需求，點「**1. 有意見（轉送簽客戶）**」
- 系統驗證：所有 Stage 1 項目打勾後才允許推進

### Stage 2：校對與審核準備
- **操作角色**：客戶
- 客戶收到通知、進入系統校審圖面，可標註留言
- 完成後點「**2. 無意見（完稿確認, 進入 Stage 3）**」
- 若需修改，點「**1. 有意見（回送設計師討論）**」

### Stage 3：最終核准與物流
- **操作角色**：設計師
- 確認客戶已 Approved，打勾 Stage 3 清單
- 點「**2. 無意見（結案, 執行製作）**」完成流程

### 返還簽核
任何角色可點「**返還簽核（退回至前置討論）**」將階段退回一級，並自動 Email 通知對方。

## SOP 規範清單

### Stage 1（設計師填寫）

| 項目 | 說明 |
|------|------|
| 基礎規格 | 提供產品名稱/SKU、數量與材質結構 |
| 精確尺寸 | 提供確切的公制尺寸 (cm) |
| 工法定義 | 確認特殊印刷效果的具體類型與區域 |
| 格式合規 | 檔案為 AI 或 EPS 格式 |
| 字體與連結 | 文字皆已轉換為外框 (Outlines) |
| 資源齊全 | 外部連結資源皆已正確嵌入或附上 |
| 除錯掃描 | 已完成基本拼寫檢查 |
| 刀模防呆 | 新刀模已標明英吋與公分雙軌尺寸 |
| 結構防呆 | 客戶已提供新結構的參考圖片或例子 |

### Stage 2（設計師填寫）

| 項目 | 說明 |
|------|------|
| 單一事實 | 確認以客戶提供的「最終檔案」為唯一生產依據 |
| 雲端同步 | 最終生產檔案已上傳至指定平台NAS |
| 權限開通 | 確認雙方對共享連結皆具備存取或編輯權限 |
| 防禦性打樣 | 已將校對稿轉為 PDF 格式回傳給客戶 |
| 微調告知 | 已明確列出為符合印刷要求所做的任何微小調整 |

### Stage 3（設計師填寫）

| 項目 | 說明 |
|------|------|
| 絕對核准 | 已收到客戶明確回覆 Approved |
| 物流盤點 | 確認特殊分批運送或免簽收等要求 |
| 物流雙重確認 | 特殊物流需求已再次向客戶確認並註記 |

> **送件規則**：所有項目必須打勾（`true`），系統才允許推進至下一 Stage。

## 標註工具

在設計圖上可留下兩種標註：

### 圖釘（Pin）
- 選擇圖釘模式後，點擊圖面任意位置放置圖釘
- 輸入留言後送出
- 適合標記特定點的問題

### 框選（Box）
- 選擇框選模式後，拖拉選取圖面區域
- 輸入對應的修改意見
- 適合標記大範圍或區塊問題

### 管理留言
- 留言顯示於右側「討論與留言」Tab，依時間排序
- 可將留言標記為「**已解決**」（變灰色）
- 可撤銷「**已解決**」狀態
- 觀察者無法新增/修改留言

## 版本管理

每次上傳新稿件會建立新版本號（V1, V2, V3...）：

- 頂部工具列點擊版本號切換顯示
- 可**刪除**任一版本（最後一版不可刪）
- 版本刪除後自動重新編號
- 每個版本獨立儲存自己的標註留言

---

## 版本差異比對

點擊「**版本比較**」進入比對模式：

1. 左右各選擇一個版本
2. 兩張設計圖並排顯示
3. 點擊「**Diff 比對**」自動計算差異（紅色標示不同處）
4. 點擊「**退出比對**」返回一般模式

## 測量工具

點擊頂部工具列「**測量工具**」圖示開啟：

- **比例設定**：輸入 mm/% 換算比例（依圖面實際尺寸校準）
- **測量操作**：在設計圖上拖拉出測量線
- **顯示結果**：距離、角度、X 軸、Y 軸分量
- **精確度切換**：0.1 / 0.01 精度
- **單位**：mm 或 in
- **CLEAR**：清除當前測量線

> 開啟測量工具期間會隱藏兩側面板，最大化圖面空間。

## 通知與 Email

系統使用 **EmailJS** 發送通知，所有推進動作都會觸發 Email：

| 動作 | 收件人 | 內容 |
|------|--------|------|
| Stage 1 → Stage 2 | 客戶 Email | 審核請求通知含專案連結 |
| Stage 2 → Stage 3 | 設計師 Email | 完稿確認通知 |
| 有意見/討論 | 客戶 Email | 討論提問通知 |
| 返還簽核 | 前置角色 Email | 退回通知含重新送審說明 |

- 系統彈出 Modal 確認收件人 Email 後才發送
- 若 API 未連線，會顯示「系統模擬送出」提示

## 分享連結

點擊「**分享連結**」複製連結，對方可直接透過連結開啟該檔案的校稿頁面，選擇角色後直接定位到對應稿件（Deep Linking 支援）。

## 資料儲存機制

| 環境 | 儲存方式 |
|------|---------|
| **Firebase 連線正常** | 雲端即時同步，跨裝置共用資料 |
| **Firebase 無法連線** | 自動 fallback 至瀏覽器 LocalStorage |

> Firebase 使用 Realtime Database，變更會即時推播給同時在線的其他使用者。
